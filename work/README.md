# work/

Planning documents, campaign logs, research notes and design explorations.
These are working artifacts — the specs and running logs behind features as
they were built — kept for reference and archaeology. They are not part of
the documentation set: user-facing docs live in the root README, `docs/`,
and the per-crate READMEs; process docs of record are
`VERSIONING.md`, `CONTRIBUTING.md` and `SECURITY.md` at the repo root.

Code comments that cite one of these documents by name (e.g. a test header
pointing at its spec, or `hypercube.rs` citing `Pnp.lean`) refer to
`work/<NAME>`.
