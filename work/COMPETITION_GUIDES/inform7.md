# Inform 7

Source: <https://ganelson.github.io/inform-website/doc/> · Project: <https://ganelson.github.io/inform-website/>

**The closest spiritual comparable to LOGOS:** a natural-language-based programming language (for
interactive fiction). How it documents an English-like language is directly instructive for us.

## Structure — TWO complementary books

1. **Writing with Inform** — a *tutorial*, "building familiarity, one new topic at a time."
   Sequential, concept-by-concept (the conventional shape, like our guide today).
2. **The Inform Recipe Book** — organized **by what you want to build / simulate**, not by language
   feature: money, clothing, liquids, conversation, breakable glass, the human body, …
   A goal-indexed cookbook that sits *beside* the tutorial.

Together they ship **~500 worked examples**, with **both alphabetical and thematic indexing** so a
reader can navigate by topic *or* by goal. Both books are bundled in every download and run inside
the app, where examples **execute in real time** as you read.

## Standout pedagogical / QOL features

- **The Recipe-Book axis.** Two orthogonal organizations of the same material: "learn the language"
  (tutorial) AND "build the thing" (recipes). For a natural-language language this matters more than
  usual — readers arrive with a *goal in English*, not a feature name.
- **~500 examples**, each cross-referenced and dual-indexed (alphabetical + thematic).
- **Principle of least surprise** as an explicit design tenet: the natural-language syntax is built so
  the phrasing a writer *guesses* is usually the phrasing that works — and the docs teach the few
  places it isn't. (See the Dorophone essay "On Inform 7, Natural Language Programming and the
  Principle of Least Surprise": <https://procyonic.org/blog/on-inform-7-natural-language-programming-and-the-principle-of-least-surprise/>.)
- Documentation written for *writers as well as programmers* — the prose register matches the
  audience, not a compiler manual.
